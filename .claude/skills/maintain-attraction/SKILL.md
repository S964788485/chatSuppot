## 工作流程

### 第一步：收集内容
使用 AskUserQuestion 询问用户要添加的内容类型：

1. **内容类型**：
   - 理论知识（概念、原理、法则）
   - 实战技巧（具体话术、操作方法）
   - 案例分析（成功/失败案例）
   - 其他（请说明）

2. **内容描述**：
   让用户提供具体的内容、笔记或要点

### 第二步：识别所属模块

根据内容的关键词和主题，判断属于哪个模块：

**识别关键词**：
- "价值"、"自我"、"认清" → 01-认清自己
- "展示"、"提升"、"沉没成本"、"筛选" → 02-提升自己
- "心态"、"下头"、"需求感"、"男神" → 03-心态建设
- "IOI"、"信号"、"兴趣"、"读懂" → 04-读懂她
- "开场"、"话题"、"聊天法"、"基础" → 05A-基础聊天技巧
- "神秘"、"角色扮演"、"离场"、"中级" → 05B-中级聊天技巧
- "推拉"、"废物测试"、"服从性"、"高级" → 05C-高级聊天技巧
- "邀约"、"场景"、"案例"、"实战" → 05D-实战应用

### 第三步：读取目标文件

使用 Read 工具读取目标文件，了解现有结构和内容。

### 第四步：整理并添加内容

**如果有对应模块**：
1. 分析文件结构，找到合适的插入位置
2. 按照文件的格式风格整理内容
3. 使用 Edit 工具将内容添加到合适位置
4. 如果需要新增子章节，遵循现有的标题层级

**如果没有对应模块**：
1. 询问用户新模块的名称和描述
2. 创建新的 .md 文件
3. 按照现有文件的格式结构编写内容

### 第五步：同步更新 SKILL.md

如果新增了模块或改变了文件结构，需要同步更新 `.claude/skills/girlfriend-chat/SKILL.md` 中的知识体系总览部分。

## 内容整理原则

1. **保持格式一致**：遵循现有文件的 markdown 格式和标题层级
2. **使用简洁语言**：避免冗长，用要点和案例说话
3. **添加篇章总结**：如果是新模块，在开头添加篇章总结
4. **核心逻辑标注**：重要概念用"核心逻辑"或"核心认知"标注
5. **案例配对话**：实战技巧需要配上对话示例
6. **表格整理对比**：适合用表格对比的内容用表格
